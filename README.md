# System Design Practice

This repository contains my weekly system design practice as I work towards becoming a strong backend and distributed systems engineer.

---

## Goals

- Build real-world system design intuition
- Understand scalability, reliability, and tradeoffs
- Prepare for senior backend/system design interviews
- Document designs clearly like production-ready systems

---

## What You'll Find Here

Each folder represents one system design problem and includes:

- Requirements (Functional + Non-functional)
- High-level architecture
- API design
- Database schema
- Scaling strategies
- Tradeoffs & decisions
- System diagrams
